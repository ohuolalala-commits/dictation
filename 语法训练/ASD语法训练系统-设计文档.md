## 1. ASD 学习者特点与语法习得障碍

### 1.1 核心挑战

| 挑战维度 | 具体表现 | 对语法学习的影响 |
|---------|---------|----------------|
| **执行功能弱** | 难以规划、切换任务、抑制冲动反应 | 不能同时处理多个语法规则；需要一次只学一个点 |
| **工作记忆有限** | 信息保持时间短，容易丢失 | 句子长了就忘记开头；需要超短句起步 |
| **抽象思维困难** | 偏好具体、可视化的信息 | "时态""语序"等抽象概念无法直接理解，必须有视觉锚点 |
| **泛化能力弱** | 学会了"A is a cat"不等于会了"B is a dog" | 每种句型模式必须**显式教过**才能迁移 |
| **感官敏感** | 对颜色、声音、动画敏感 | 界面必须极简、安静、可预测 |
| **焦虑高发** | 害怕犯错，一次失败可能拒绝继续 | 零负面反馈；错误只提示不批评 |
| **刻板/重复行为** | 可能反复做同一题不动 | 系统需内置"温和推进"机制 |
| **语言处理差异** | 语法性别、功能词（the/a/is）常被忽略 | 功能词需要**高亮标记**才能被注意到 |

### 1.2 ASD 学习者的优势（可利用）

- ✅ **模式识别能力强**：一旦建立"规则→例句→视觉标记"的映射，识别速度可能超常
- ✅ **喜欢重复和可预测性**：恰好匹配语法练习需要的大量重复
- ✅ **对规则忠诚**：学会规则后会严格应用，不容易"随性"犯错
- ✅ **视觉学习偏好**：色彩编码、图式、表格是天然的教学工具
- ✅ **专注力（感兴趣时）**：对特定主题的持续注意力可能远超常人

## 2. 设计核心原则

### 原则一：一次一个语法点（One Rule at a Time）

```
❌ 错误做法：同时教 am/is/are + 人称代词 + 形容词
✅ 正确做法：
   第1关：只练 "I am ___"（I am happy / I am a boy / I am tall）
   第2关：只练 "He is ___" + "She is ___"
   第3关：只练 "You are ___" + "They are ___"
   第4关：混合练习 am/is/are（此时每个单项都已自动化）
```

### 原则二：视觉语法标记（Visual Grammar Markers）

每种语法成分有**固定颜色**，整个系统保持一致：

| 语法成分 | 颜色 | 色值 | 说明 |
|---------|------|------|------|
| 主语（名词/代词） | 🔵 蓝色 | `#4A90D9` | 谁/什么 |
| Be 动词 (am/is/are) | 🟢 绿色 | `#5CB85C` | 连接词 |
| 动作动词 | 🟠 橙色 | `#F0AD4E` | 做什么 |
| 功能词 (the/a/can/in/on) | 🟣 紫色 | `#9B59B6` | 小帮手词 |
| 时间标记 | 🔴 粉色 | `#E91E63` | 什么时候 |
| -ing / -s / -ed（屈折词缀） | 🟡 黄色加粗 | `#FFC107` | 词尾巴变化 |

### 原则三：具体先于抽象（Concrete Before Abstract）

```
步骤1：图片 + 例句（完全具体）  → 🧒看图："He is running."
步骤2：例句 + 色块标记           → He 🟢is 🟠running.
步骤3：填空（视觉提示保留）      → He ___ running. [is / am / are] 色块保留
步骤4：填空（提示渐隐）          → He ___ running. [is / am / are] 无颜色
步骤5：自主产出                  → 看图，自己写出 "He is running."
```

### 原则四：零负面反馈（Emotional Safety）

```
❌ 绝对不用：  "错了！" "不对！" "❌" 红色叉号 "再想想"（有压力）
✅ 只用：
   - 正确：⭐ "Good job!"（绿色勾 + 柔和音效）
   - 不正确：💡 自动显示正确答案（黄色高亮 1.5 秒），然后进入下一题
   - 不评价、不对比、不排名
   - 没有"分数"，只有"🌟 星星收集"
```

### 原则五：超量重复 + 间隔复习（Massed → Spaced）

```
新语法点：当天 8-12 次同一模式的重复（massed practice）
     ↓ 正确率 ≥ 85%
间隔1天：4 题复习
     ↓ 正确率 ≥ 85%
间隔3天：3 题复习
     ↓ 正确率 ≥ 85%
间隔7天：2 题复习 → 标记为"已掌握"
```

### 原则六：可预测的流程（Predictable Routine）

每个关卡的结构**完全相同**，只是语法内容不同。ASD 学生一旦熟悉流程，认知负荷大幅降低。

## 3. 语法等级体系（8 级 24 关）

```
Level 1 🟢 Be 动词基础
├── 1-1  I am ...      （身份：I am a student.）
├── 1-2  He/She is ... （第三人称单数：He is tall.）
└── 1-3  You/We/They are ... （复数/第二人称）

Level 2 🟣 冠词 + 名词复数
├── 2-1  a vs an       （a cat / an apple）
├── 2-2  a/an vs the   （泛指 vs 特指，配合图片）
└── 2-3  名词复数 -s/-es/-ies （one cat → two cats）

Level 3 🟠 一般现在时（无三单）
├── 3-1  I/You/We/They + 动词原形 （I eat breakfast.）
├── 3-2  否定：don't + 动词原形   （I don't like fish.）
└── 3-3  疑问：Do you...?        （Do you like dogs?）

Level 4 🟡 一般现在时（三单）
├── 4-1  He/She/It + 动词+s      （He eats breakfast.）
├── 4-2  否定：doesn't + 动词原形 （She doesn't like fish.）
└── 4-3  疑问：Does he...?       （Does he like dogs?）

Level 5 🟣 情态动词 Can
├── 5-1  I can / I can't         （I can swim. / I can't fly.）
├── 5-2  Can you...? 问答        （Can you swim? Yes, I can.）
└── 5-3  He can / She can        （He can run fast.）

Level 6 🟠 现在进行时
├── 6-1  I am + verb-ing         （I am reading.）
├── 6-2  He/She is + verb-ing    （She is sleeping.）
└── 6-3  They are + verb-ing     （They are playing.）

Level 7 🟣 介词（方位）
├── 7-1  in / on / under         （The cat is in the box.）
├── 7-2  next to / behind / in front of
└── 7-3  综合：There is/are + 介词 （There is a book on the desk.）

Level 8 🔵 Have/Has
├── 8-1  I/You/We/They have      （I have a dog.）
├── 8-2  He/She/It has           （She has a cat.）
└── 8-3  综合复习（Level 1-8 混合）
```

### 为什么选这 8 个语法点？

| 筛选标准 | 说明 |
|---------|------|
| **高频率** | 这 8 个点覆盖日常英语 70%+ 的句子结构 |
| **可视化** | 每个点都有对应的图片/场景，不依赖抽象解释 |
| **递进依赖** | L1→L2→L3→L4 有严格的先修关系 |
| **可分段** | 每个点拆成 3 关，每关 5-8 分钟完成 |
| **有终点感** | 24 关的进度条让 ASD 学生感到"可以完成" |

## 4. 教学模式：五步法

每关内部采用**固定的五步教学法**，ASD 学生一旦适应流程，就不再需要理解"新指令"。

```
┌─────────────────────────────────────────────────┐
│  Step 1: SEE 👀   看例句 + 图片（10-15秒）        │
│  Step 2: COLOR 🎨 色块标注语法结构（被动观察）     │
│  Step 3: MATCH 🔗  选择填空（2选1，提示全开）      │
│  Step 4: BUILD 🧱  组句（拖拽词块，提示渐隐）      │
│  Step 5: USE ✏️   看图写句（独立产出，仅图片提示） │
└─────────────────────────────────────────────────┘
```

### Step 1: SEE — 看例句

- **展示**：一张清晰的图片 + 一个正确的英文句子
- **TTS 朗读**：慢速 0.4x 播放句子（flutter_tts）
- **时长**：10-15 秒自动过渡，也可点击"再看一次 🔁"
- **目的**：建立"画面 → 英文句子"的直接映射，不解释规则

```
示例（Level 1-1: I am）：

┌──────────────────────────┐
│                          │
│   🧒（男孩图片）          │
│                          │
│   I am a boy.            │
│                          
│   🔊 点击听              │
│                          │
└──────────────────────────┘
```

### Step 2: COLOR — 色块标注

- **展示**：与 Step 1 相同的图片 + 句子，但加上颜色标记
- **动画**：颜色块逐个**缓慢出现**（每个 0.8 秒），不是同时弹出
- **目的**：引导学生**注意功能词**（ASD 学生常跳过 is/the/a）

```
示例：

   🔵I  🟢am  🟣a  🔵boy.
   (蓝色) (绿色) (紫色) (蓝色)

颜色逐个出现 →
   第1秒: 🔵I
   第2秒: 🔵I 🟢am
   第3秒: 🔵I 🟢am 🟣a
   第4秒: 🔵I 🟢am 🟣a 🔵boy.
```

### Step 3: MATCH — 选择填空

- **形式**：2 选 1 或 3 选 1
- **提示等级**：4 级可调（见第 10 节渐隐引擎）
- **错误处理**：选错后自动显示正确答案（黄色闪烁 1.5 秒），不评价，自动进入下一题
- **正确反馈**：绿色边框 + ⭐ + 柔和叮咚声

```
示例（Level 1-1: I am）：

   I ___ a student.

   [ am ]   [ is ]    ← 两个按钮，色块可渐隐
```

### Step 4: BUILD — 拖拽组句

- **形式**：屏幕底部有乱序词块，拖到上方横线组成句子
- **词块颜色**：与 Step 2 的颜色编码一致
- **提示等级**：可渐隐（先有色块 → 后无色块 → 最后连词序提示也没有）

```
示例：

目标句式：I am happy.

底部词块（乱序）：
┌────┐ ┌────┐ ┌────┐
│ am │ │  I │ │happy│
└────┘ └────┘ └────┘

上方放置区：
[____] [____] [____]
   1      2      3
```

### Step 5: USE — 看图写句

- **形式**：只给图片，不给任何文字提示
- **输入**：软键盘输入完整句子
- **宽松评分**：大小写不扣分、末尾句号不扣分、多一个空格不扣分
- **通过标准**：核心语法结构正确即可

```
示例：

┌──────────────────────────┐
│   🌳（一棵树的图片）       │
│                          │
│   请写出完整的句子：       │
│                          │
│   [__________________]   │
│                          │
│   提示按钮（可渐隐）：     │
│   💡 It / is / tree      │
└──────────────────────────┘
```

## 5. UI/UX 设计规范（ASD 语法专用）

### 5.1 色彩系统

基于 SpeakStep 已有的 ASD 配色，语法系统追加：

```
背景：      #FAF8F5（暖米白）
卡片：      #FFFFFF（纯白）
主文字：    #333333（深灰，非纯黑）
语法高亮：  见第2节色表（蓝绿橙紫粉黄）
正确反馈：  #E8F5E9（极浅绿底）+ #4CAF50（绿色勾）
提示显示：  #FFF8E1（极浅黄底）+ #F9A825（黄色边框）
禁用/灰色： #E0E0E0
```

### 5.2 禁止事项

| ❌ 禁止 | 原因 |
|--------|------|
| 红色错误标记 | 引发焦虑 |
| 计时器/倒计时 | 时间压力导致崩溃 |
| 分数/百分比排名 | 社交比较焦虑 |
| 弹窗/模态框 | 打断心流 |
| 自动播放音频 | 感官过载 |
| 闪烁动画 | 可能诱发癫痫/不适 |
| 卡通人物表情变化 | 难以解读面部表情 |
| "游戏化"音效（升级音/失败音） | 感官刺激过度 |
| 多于 3 个选项 | 选择 overload |
| 滚动页面 | 空间定位困难 |

### 5.3 每关的页面结构（固定不变）

```
┌─────────────────────────────────┐
│  ← 退出     Level 1-1     ⭐×12 │  ← 顶部栏（40px）
│            I am ...             │
├─────────────────────────────────┤
│  ●●●○○○○○  进度: 3/8           │  ← 进度条（8px）
├─────────────────────────────────┤
│                                 │
│      [当前题目区域]              │  ← 主体（占 65% 屏幕）
│      图片 + 句子/填空/拖拽       │
│                                 │
├─────────────────────────────────┤
│  💡 提示（可选，渐隐中）         │  ← 提示区（占 10%）
├─────────────────────────────────┤
│  [选项A]        [选项B]         │  ← 操作区（占 20%）
│         🔊 播放                 │
└─────────────────────────────────┘
```

### 5.4 过渡动画规范

```
允许：
✅ 淡入（fade in, 300ms）
✅ 轻微缩放（scale 0.95 → 1.0, 200ms）
✅ 颜色平滑过渡（background-color transition 400ms）
✅ 词块滑动到位（translate, 250ms, ease-out）

禁止：
❌ 弹跳（bounce）
❌ 旋转（rotate）
❌ 闪烁（blink / flash）
❌ 粒子/彩带/烟花效果
❌ 任何小于 200ms 的突然变化
```

## 6. Level 1–2 详细教案

### Level 1-1: I am ...（身份与感受）

**教学时长**: 8-12 分钟 | **题目数**: 8 题

#### 词汇准备（本关会出现的词）

| 类别 | 词汇 |
|-----|------|
| 身份 | a boy, a girl, a student, a teacher |
| 感受 | happy, sad, hungry, tired |
| 描述 | tall, short, big, small |

#### 教学流程

**Step 1 SEE（2题 × 15秒）**

```
题1:  🧒男孩图片  →  "I am a boy."    🔊
题2:  😊笑脸图片  →  "I am happy."    🔊
```

**Step 2 COLOR（2题 × 20秒，颜色逐个出现）**

```
题3:  🧒男孩图片  →  🔵I 🟢am 🟣a 🔵boy.
题4:  😊笑脸图片  →  🔵I 🟢am 🔵happy.
```

> 教师/家长旁白（可选）：
> "看到绿色的 am 了吗？I 后面总是跟着 am。I am, I am, I am。"

**Step 3 MATCH（4题，2选1）**

```
题5:  🧒男孩图片  →  I ___ a boy.     [ am ✅ / is ]
题6:  😊笑脸图片  →  I ___ happy.     [ am ✅ / are ]
题7:  😢难过图片  →  I ___ sad.       [ is / am ✅ ]
题8:  🧑学生图片  →  I ___ a student. [ are / am ✅ ]
```

**Step 4 BUILD（4题，拖拽组句）**

```
题9:  拖拽: [am] [I] [happy]     →  目标: I am happy.
题10: 拖拽: [a] [am] [I] [boy]   →  目标: I am a boy.
题11: 拖拽: [tall] [I] [am]      →  目标: I am tall.
题12: 拖拽: [hungry] [am] [I]    →  目标: I am hungry.
```

**Step 5 USE（4题，看图写句）**

```
题13: 🧒 → 学生写出: ______________   (I am a boy.)
题14: 😊 → 学生写出: ______________   (I am happy.)
题15: 😢 → 学生写出: ______________   (I am sad.)
题16: 📏高个子→ 学生写出: ___________ (I am tall.)
```

#### 渐隐策略（Level 1-1）

```
题 5-6:  选项按钮保留颜色标记  [🟢am] [is]
题 7-8:  选项按钮去掉颜色       [am] [is]
题 9-10: 拖拽词块保留颜色       🟢am 🔵I 🔵happy
题 11-12: 拖拽词块去掉颜色      am I tall
题 13-14: 有 💡提示按钮（点击显示 I / am / happy 三个词）
题 15-16: 无提示
```

### Level 1-2: He is / She is ...

**教学时长**: 8-12 分钟 | **题目数**: 8 题

#### 核心区别

本关的关键教学点是：**I am → He is / She is 的动词变化**。ASD 学生需要明确看到这个"am 变成 is"的过程。

#### 对比引入法（本关特有）

在 Step 1 和 Step 2 之间，插入一个**"看变化"环节**：

```
屏幕同时展示两张卡片：

左边：🧒 I am a boy.       右边：👦 He is a boy.
      🔵I 🟢am 🟣a 🔵boy        🔵He 🟢is 🟣a 🔵boy.

关键：am 和 is 都用绿色闪烁 2 秒，让学生注意到"词变了"
```

#### 教学流程

**Step 1 SEE（2题）**

```
题1: 👦（男孩指着另一个男孩） → He is a boy.    🔊
题2: 👧（女孩指着另一个女孩） → She is a girl.   🔊
```

**Step 1.5 对比（2题，新环节！）**

```
题3: 🧒 I am happy.    →   👦 He is happy.   （观察 am→is）
题4: 🧒 I am tall.     →   👧 She is tall.   （观察 am→is）
```

**Step 2 COLOR（2题）**

```
题5: 👦 → 🔵He 🟢is 🔵tall.
题6: 👧 → 🔵She 🟢is 🔵happy.
```

**Step 3 MATCH（4题，二选一：am vs is）**

```
题7:  👦 He ___ a boy.      [ am / is ✅ ]
题8:  👧 She ___ a girl.    [ am / is ✅ ]
题9:  👦 He ___ tall.       [ is ✅ / am ]
题10: 👧 She ___ happy.     [ am ✅→ 故意，让学生发现不对 ]
                              → 选 am 后，自动显示 is，不评价
```

> 题10 是故意的"陷阱题"，但不叫陷阱，叫"找不同"。
> 选 am 后系统不批评，只是把 is 高亮 1.5 秒，然后继续。

**Step 4 BUILD（4题，拖拽）**

```
题11: 拖拽 [is] [He] [tall]       → He is tall.
题12: 拖拽 [happy] [She] [is]     → She is happy.
题13: 拖拽 [a] [is] [He] [boy]   → He is a boy.
题14: 拖拽 [girl] [She] [is] [a] → She is a girl.
```

**Step 5 USE（4题，看图写句）**

```
题15: 👦 → _______________  (He is a boy.)
题16: 👧 → _______________  (She is a girl.)
题17: 👦😊 → _____________  (He is happy.)
题18: 👧📏 → _____________  (She is tall.)
```

### Level 1-3: You are / We are / They are

**与前两关的关键区别**：am → are 的过渡，以及**人称与动词的对照表**。

#### 对照表（固定展示在屏幕上方）

```
┌────────────────────────────────┐
│  I     →  am                   │
│  He    →  is                   │
│  She   →  is                   │
│  You   →  are  ← 新学的！      │
│  We    →  are  ← 新学的！      │
│  They  →  are  ← 新学的！      │
└────────────────────────────────┘
```

这张表在 Level 1-3 的所有题目中**始终可见**（不渐隐），作为"认知支架"。

#### 教学流程（简化描述）

**Step 1 SEE（3题）**
```
题1: 👥两个人 + 指向学生 → You are students.  🔊
题2: 👥一群人指着自己    → We are happy.      🔊
题3: 👥多个人           → They are tall.      🔊
```

**Step 2 COLOR（3题）**
```
题4: 🔵You 🟢are 🔵students.
题5: 🔵We 🟢are 🔵happy.
题6: 🔵They 🟢are 🔵tall.
```

**Step 3 MATCH（6题，3选1：am/is/are）**
```
题7-12: 不同人称代词 + ___ + 形容词/名词，选 am/is/are
```

**Step 4 BUILD（6题）**

**Step 5 USE — 混合关（6题）**

本关 Step 5 混合 I am / He is / She is / You are / They are 全部模式。

### Level 2-1: a vs an

**核心规则（显式教）**：
- a + 辅音开头 → a cat 🐱
- an + 元音开头 → an apple 🍎

**ASD 教学方法**：不解释"元音/辅音"术语，而是用"嘴巴张开的声音"。

#### 视觉系统

```
所有 a/an 题目中：
- 单词首字母如果是 a/e/i/o/u → 用 🔴红色 标记
- 单词首字母如果是其他字母 → 用 ⚫灰色 标记

示例：
🍎 🔴apple  →  an apple   （红色提醒：元音开头！）
🐱 ⚫cat    →  a cat       （灰色：辅音开头）
```

#### 3 关递进

```
2-1 (本关): a/an 选择（给图片+单词，选 a 或 an）
2-2:        a/an vs the 区分（泛指 vs 特指，用图片对比）
2-3:        名词复数 -s / -es / -ies（给单数，写复数）
```

## 7. Level 3–4 详细教案

### Level 3-1: I/You/We/They + 动词原形（一般现在时）

**关键设计**：引入"时间圈"概念。

#### 时间圈视觉

```
屏幕左侧固定显示：

    ┌──────────┐
    │  🌅       │
    │  每天     │  ← "一般现在时 = 每天做的事"
    │  every    │
    │  day      │
    └──────────┘

配合 7 张日常活动图：eat breakfast, go to school, play, sleep, drink water, read, run
```

#### 核心句型

所有题目的主语只使用 I / You / We / They（刻意排除 He/She/It，等到 L4 再引入）。

```
I eat breakfast.    🍳
You go to school.   🏫
We play soccer.     ⚽
They read books.    📚
```

#### 否定引入（3-2）

```
肯定: I like fish.    🐟😊
否定: I don't like fish.  🐟😐（配图：摇头 + 推开的鱼）

don't = do + not 的合体（配图：两个词块合并成一个的动画）
```

#### 疑问引入（3-3）

```
陈述: You like dogs.   🐕
疑问: Do you like dogs?  🐕❓

配图：陈述句末尾的句号 → 变成问号；Do 从句子外面"跳"到最前面
动画：Do 词块从左侧滑入 → 句号变问号
```

---

### Level 4: 三单（最关键也是最难的 Level）

**为什么三单对 ASD 学生特别难？**
- 需要同时处理：主语识别 + 动词变形 + 发音变化
- -s / -es / -ies 三种形式需要分别学
- 否定和疑问中的 doesn't / Does 更是认知 overload

#### 解决方案：分三步，每步只加一个变量

**4-1: He/She/It + 动词+s（纯肯定句）**

```
Step 1 SEE:    👦🍳  He eats breakfast.    （eat → eats，-s 用黄色加粗）
Step 2 COLOR:  🔵He 🟠eats 🟡-s 🔵breakfast.
Step 3 MATCH:
    He ___ breakfast.     [ eat / eats ✅ ]
    She ___ to school.    [ go / goes ✅ ]   ← 注意 goes 是不规则！
    It ___ water.         [ drink / drinks ✅ ]
```

**关键：-s 的黄色标记**

```
动词 + -s 永远把 -s 标成 🟡黄色加粗：
   eat     → eat🟡s
   go      → go🟡es
   drink   → drink🟡s
   watch   → watch🟡es
   fly     → fl🟡ies  （y→i+es 整个变黄）

ASD 学生看到黄色出现，就知道"主语是 he/she/it"
```

**4-2: doesn't（否定）**

> 教学点：doesn't = does + not，且 doesn't 后面的动词**不要**加 -s

```
肯定: He eats fish.    🐟😊
否定: He doesn't eat fish.  🐟😐

关键视觉：
   He eats fish.     → eats 有🟡s
   He doesn't eat fish. → eat 没有 s！🟡消失了！

动画：-s 从 eat 上"飞走"，因为 doesn't 已经包含了"三单信息"
```

这个动画极其重要——ASD 学生通过视觉变化理解"doesn't 偷走了 -s"。

**4-3: Does...?（疑问）**

```
陈述: He likes dogs.   🐕
疑问: Does he like dogs?  🐕❓

同样：likes 的 s → like（s 被 Does "偷走"了）
```

## 8. Level 5–6 详细教案

### Level 5: Can / Can't

**为什么 can/can't 放在前面？**
- 极度高频
- 动词形式无变化（can + 动词原形，永远不变）
- 含义非常具体（能力 = 能做到/做不到）
- 视觉化最容易：✅能做到 ❌做不到

#### 图片选择

```
✅ 能做到的事（真实照片级图片）：
   swim 🏊, run 🏃, jump 🤾, read 📖, cook 🍳, sing 🎤, dance 💃

❌ 做不到的事（幽默但不荒谬）：
   fly 🕊️, swim in the sky, eat a mountain, run on water
```

#### 否定特别设计

```
5-1: I can ___.  /  I can't ___.
5-2: Can you ___?  →  Yes, I can. / No, I can't.
5-3: He can ___.  /  She can't ___.
```

### Level 6: 现在进行时（Be + -ing）

**为什么放在一般现在时之后？**
- 需要先掌握 am/is/are（L1）
- 需要先掌握动词原形（L3）
- 现在进行时 = L1（be动词）+ L3（动词）+ -ing

#### 核心视觉

```
现在进行时 = "正在做"  →  🕐 时间圈标为"now/right now"

动词 + -ing 用 🟡黄色（与三单 -s 同色系，延续"词尾变化 = 黄色"的规则）：

   I 🟢am 🟠read🟡ing.
   He 🟢is 🟠runn🟡ing.  （注意双写 n）
   They 🟢are 🟠play🟡ing.
```

#### -ing 拼写变化（分三小步教）

```
6-1: 一般动词 +ing      → eat→eating, play→playing, read→reading
6-2: 去 e +ing          → make→making, write→writing, dance→dancing
6-3: 双写 +ing          → run→running, swim→swimming, sit→sitting
```

## 9. Level 7–8 详细教案

### Level 7: 介词（方位）

**ASD 优势利用**：空间关系对许多 ASD 学生是具象的、可视觉化的。

#### 核心设计：一张图贯穿始终

所有题目使用**同一张桌子图片** + 一只小猫在不同位置：

```
🐱📦（猫在盒子里）     → The cat is in the box.
🐱📦（猫在盒子上）     → The cat is on the box.
🐱📦（猫在盒子下）     → The cat is under the box.
🐱📦（猫在盒子旁边）   → The cat is next to the box.
🐱📦（猫在盒子后面）   → The cat is behind the box.
🐱📦（猫在盒子前面）   → The cat is in front of the box.
```

#### 介词用紫色标记

所有介词统一用 🟣紫色：
```
The cat is 🟣in the box.
The cat is 🟣on the box.
The cat is 🟣under the box.
```

#### 三关递进

```
7-1: in / on / under（3 个最基础介词，空间关系明确）
7-2: next to / behind / in front of（需要更多参照点判断）
7-3: There is/are + 介词（引入存在句）
     There is a cat under the table.
     There are two cats on the chair.
```

---

### Level 8: Have / Has

**have/has 的教法核心**：
- 学生已经通过 L1 掌握了 am/is/are 的人称对应
- 现在只需要类比：am/is/are → have/has

#### 类比表

```
┌──────────────────────────────────────┐
│  Be 动词（你已经会了！）              │
│  I am    He is    She is    You are  │
│  ─────────────────────────────────── │
│  Have/Has（新的！但规则很像）         │
│  I have  He has   She has  You have │
│                                      │
│  👉 有 s 的都是 he/she！             │
│     is → has  都有 s！               │
└──────────────────────────────────────┘
```

## 10. 渐隐提示引擎（语法版）

沿用 SpeakStep D 模块的 4 级渐隐，适配语法训练：

### 10.1 四级提示定义

| 级别 | 名称 | 效果 | 触发条件 |
|-----|------|------|---------|
| **L4** | 全提示 | 选项保留颜色标记 + 语法对照表可见 + 💡按钮亮 | 新语法点前 3 题 |
| **L3** | 半提示 | 选项颜色去掉 + 对照表仍可见 + 💡按钮亮 | 连续 2 题正确后降级 |
| **L2** | 弱提示 | 对照表部分隐藏（折叠）+ 💡按钮暗（仍可点击） | 连续 2+2=4 题正确后降级 |
| **L1** | 无提示 | 无颜色、无对照表、无💡按钮 | 连续 6 题正确后降级 |
| **L0** | 独立 | 完全独立产出（Step 5 USE 模式） | 本关最终目标 |

### 10.2 升级/降级规则

```
初始: L4（全提示）

如果连续 2 题正确（匹配模式）:
    L4 → L3
    L3 → L2
    L2 → L1
    L1 → L0（独立）

如果连续 2 题错误:
    L1 → L2
    L2 → L3
    L3 → L4
    L4 → 保持 L4 + 额外展示正确答案 3 秒

特殊规则：
- 升级只发生在同一题型内（MATCH 内部升级，不跨到 BUILD）
- 换题型时重置为 L3（因为 BUILD 比 MATCH 难）
- L0 只适用于 Step 5 USE
```

### 10.3 对照表的渐隐

Level 1-3 的人称-动词对照表有自己的渐隐节奏：

```
第 1-3 题：表格完整可见，当前行 🟡高亮
第 4-6 题：表格仍可见，但高亮减弱为浅灰
第 7-9 题：表格折叠为一行提示："💡 I→am  He→is  You→are"
第 10+ 题：表格完全隐藏
```
