# 开发日志 — 2026年06月21日

## ✅ 完成事项

### Phase 1: 文件架构重构

1. **单文件拆分**：将 282KB 的 `index.html` 拆分为模块化结构
   - `css/theme.css` — 所有暗金埃及风样式（含新增登录浮层样式）
   - `js/wordbanks.js` — 8个词库数据（约 205KB）
   - `js/dictation-core.js` — 核心默写逻辑（~600行）
   - `index.html` 改为引用外部文件，自身缩至约 40KB
   - 备份原文件为 `index_backup_20260621.html`

2. **LeanCloud 配置**：创建 `js/leancloud-config.js`，老师只需替换 AppID/AppKey 即可启用云服务

### Phase 2: 登录系统

3. **`js/auth.js` 完整实现**：
   - `checkSession()` — 会话恢复（记住我）
   - `login(username, password)` — 学生/老师登录
   - `registerTeacher()` — 老师首次注册
   - `createStudent()` — 老师创建学生账号
   - `logout()` — 退出登录 + 强制同步
   - `getUserRole()` / `isTeacher()` / `isStudent()` — 角色判断

4. **登录浮层 UI**（暗金埃及风）：
   - 暗色半透明遮罩 + 金色卡片
   - 用户名 / 密码输入框 + 记住我复选框
   - «登录/注册»模式切换
   - 错误提示（红色，中文友好化）
   - Header 退出按钮 + 用户名标签
   - 回车键快速登录

5. **启动流程改造**：
   - `appStartup()` — 先检查登录 → 有会话自动恢复 → 无会话显示登录浮层
   - LeanCloud 未配置时以本地模式运行，不影响单机使用

### Phase 3: 云端数据同步

6. **`js/sync.js` 同步引擎**：
   - 本地优先 + 后台异步批量同步策略
   - `queueAnswerSync()` — 答题后入队
   - `queueProgressSync()` — 进度变更入队
   - `flushToCloud()` — 防抖批量写入（5秒合并）
   - `upsertStudyProgress()` — 进度 upsert
   - `upsertPerItemStatuses()` — 逐词状态批量 upsert
   - `fetchCloudProgress()` + `mergeCloudProgress()` — 登录后拉取合并

7. **改造 `dictation-core.js`**：
   - `loadWrongSet()` — 新增云端拉取合并逻辑
   - `checkAnswer()` — 正确/错误分支均新增 cloud sync hook
   - `switchBank()` — 切换前强制 flush

### Phase 4: 教师仪表盘

8. **`teacher.html` 新建**：
   - **统计卡片**：学生总数、今日答题、整体正确率、错题总数
   - **Tab 1 — 学生概览**：学生×词库矩阵表，显示进度/正确率/最后练习
   - **Tab 2 — 词条明细**：选择学生+词库 → 逐词 🟢/🔴/⬜ 状态表格，支持状态筛选
   - **Tab 3 — 答题时间线**：时间倒序流水，支持学生/词库/天数筛选
   - **Tab 4 — 学生管理**：添加学生（姓名+账号+密码）、重置密码、删除学生
   - 完整暗金埃及风 UI，响应式适配

9. **`js/teacher-dashboard.js` 完整实现**：
