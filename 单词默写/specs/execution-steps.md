# 执行步骤

## 日常开发流程

### 1. 打开项目
```
双击 C:\西安备课\单词默写\launch.pyw
```
或在浏览器中直接打开 `英语默写本.html`

### 2. 编辑代码
- 所有修改均在 `英语默写本.html` 这一个文件中
- 使用支持 HTML/CSS/JS 代码高亮的编辑器（VS Code / Cursor 推荐）

### 3. 测试验证
- 刷新浏览器页面
- 测试流程：
  1. 切换词库，确认标题和数据正确切换
  2. 输入正确/错误答案，确认反馈正常
  3. 测试错题本功能（答错 → 加入错题本 → 错题复习）
  4. 测试键盘快捷键（Enter / Ctrl+Enter）
  5. 测试键盘音效
  6. 在移动端/iPad 上测试响应式

### 4. 记录日志
- 在 `dev-logs/YYYY-MM-DD.md` 中记录今日完成事项
- 更新待办清单

## 新增词库的标准步骤

1. **添加下拉选项** — 在 `<select id="bankSelect">` 中添加一行：
   ```html
   <option value="newBankId">新词库名称</option>
   ```

2. **添加数据** — 在 `wordBanks` 对象末尾（`};` 之前）添加：
   ```javascript
   newBankId: {
     name: '名称',
     icon: '📗',
     subtitle: '副标题',
     doneMsg: '完成消息',
     items: [
       { cat:'分类', cn:'中文', en:'English', hint:'h___', tip:'提示', alt:[] }
     ]
   },
   ```

3. **适配显示模式**（如需要） — 在 `renderItem()` 中判断 `currentBankId`，自定义 `promptMain` 和 `promptCn`

4. **测试** — 切换新词库，验证所有功能正常

## 词库数据规范

### item 字段说明

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `cat` | ✅ | 分类标签 | `'Unit1·比较级'` |
| `cn` | ✅ | 中文提示（用户看到的） | `'比'` |
| `en` | ✅ | 英文正确答案 | `'than'` |
| `hint` | ✅ | 提示（首字母+长度） | `'t____'` |
| `tip` | ✅ | 知识点提示 | `'用于比较级句中'` |
| `alt` | ✅ | 可接受的其他答案（数组） | `['was','were']` |
| `base` | 仅 pastTense | 动词原型 | `'be'` |

### hint 格式规范
- 提示单词首字母，其余用 `_` 下划线表示
- 带空格的多词：`'w__/w___'`
- 长度应与正确答案匹配

## 部署

- 目标环境：Windows 桌面（通过 launch.pyw）+ iPad 浏览器（通过文件共享或本地服务器）
- 无需构建或打包，编辑后直接刷新浏览器即可生效
- 如需在 iPad 上使用，将文件复制到 iPad 或通过局域网共享
